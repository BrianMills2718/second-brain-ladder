> *"structural floor enforced, semantic correctness a calibrated gate, structure
> derived-not-drifted."*

**‚Öî true.**
- **structural floor enforced ‚Äî TRUE.** `npm run check` exits 0 and is a deploy-blocking

## The five scrutiny points

**1. Content correctness at scale ‚Äî genuinely high.** Adversarial OWL/SHACL/DL/embeddings
audit of all 100 concepts found **no clean critical category-error plants**. The
trap-prone concepts are correct *and explicitly hedged* (`universal-restriction`,
`functional-property`, `validation`/`shacl-validation`, `sameas`/`entity-resolution`,
`transe`, `description-logic`/SROIQ). Two items fixed in this PR:
- `consistency` example attributed inconsistency to *one bad triple* rather than the
  *axiom set* ‚Äî a mild OWA-trap nudge. Reworded.
- `non-monotonic-reasoning` tied non-monotonicity to OWA, but OWL/OWA entailment is
  *monotonic*; it arises from defaults/NAF. Reworded + `PREREQ_WHY` corrected.
- Caveat: this quality was verified *by review, not by the gate* (which is dormant).

**2. Gate soundness ‚Äî the weakest area.**
- **Only 3 of 8 FAIL gates have a negative fixture** (richness, band-closure,
  layout-sanity). `module-size`, `glossary-coverage`, `domain-coverage`,
  `goal‚Üíachievement`, `stage‚Üînode-parity` have **no fire-test** ‚Äî nothing proves they
  *can* fail. `MACHINERY_NEEDED.md:311` ("test-gates proving *each* FAILs its fixture")
  is false for 5/8.
- **stage‚Üînode parity essentially cannot fail** (`validate-content.mjs:457-460`): checks
  only lessons‚Üínodes, but nodes are *generated* from lessons (`graph.ts:42`). A gate that
  can't fail reads as green assurance ‚Äî worse than absent.
- **richness is a confirmed tripwire** (a 6-node hub-and-spoke passes it; real margins
  3‚Äì7√ó) ‚Äî honest in the docs, not a quality lever.
- **`prereqMinimality`/`layerConsistency` are dead code** ‚Äî `layerConsistency` imported
  nowhere; minimality prints "38 warnings", gates nothing.

**3. R10 keyless CI + self-consistency ‚Äî sound logic, weak calibration, non-blocking.**
Self-consistency is correct (2-of-3 only on a "wrong" first sample; "misleading"
advisory; parse errors ‚Üí exit 2, no vacuous pass). But the frozen set is only 3
reconstructed defects + 3 correct + 3 fixed, labels are binary, and **trap-class is never
asserted** (`v.trap` compared to nothing) ‚Äî proves "flags known-bad" but not "names the
right trap." "all 64" is stale; it runs 100. Keeping `--all` advisory is fine; the
9-case calibration regression is deterministic enough to be **blocking**.

**4. R5 generation loop unbuilt ‚Äî the crux, not a footnote.** Everything is hand-authored
against the gates. The thesis ("generate these reliably at high quality") is unproven ‚Äî
the generator doesn't exist. The gates have only faced content a careful human already
shaped to pass them, never an adversary seeking the cheap path.

**5. Docs ‚Äî SYSTEM.md sufficient; CLAUDE.md was the wrong project.** SYSTEM.md (inventory
+ runbook + honest footnotes) is good. `CLAUDE.md` was verbatim the G√∂del project's (60
concepts, a FastAPI judge, math-correctness) ‚Äî an agent following it would chase a
nonexistent backend and the wrong domain. Rewritten in this PR.

## Cross-project meta-finding
`godel-concept-ladder` and `second-brain-ladder` independently converged on the same
R1‚ÄìR14 gate taxonomy **and stalled at the same place**: gates built and self-testing,
generator unbuilt, semantic gate calibrated-but-not-gating. The convergence validates the
taxonomy; it also warns that hand-authoring keeps producing good content, which masks
that the actual deliverable (the generator) is untested.

## Highest-value next machinery
**Build the first real propose‚Üígate‚Üírevise slice (R5) ‚Äî one module, end to end.** A
generator is the adversary the gates were built for, and it will instantly find every
paper tiger (stage-parity that can't fail, dead lints, dormant R10). It ranks the gate
weaknesses by what actually breaks. Sequence:

   matter.
4. The R5 slice itself ‚Äî the real test of the whole thesis.

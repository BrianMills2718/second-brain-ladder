main line:

  gates; tension-resolution; skill-focus). Canonical, but lives in the *other* repo.
- `second-brain-ladder/docs/ENGINE.md` — the *engine* north-star ("one engine,
  curriculum-as-data, generate any goal"). **On master.**
- `docs/VISION.md`, `docs/CAPABILITY_MODEL.md`, and a second-brain `docs/METHODOLOGY.md`
  (the parallel agent's vision) — **committed on an unmerged branch, NOT on master**;
  nothing on master references them. → `orphaned-source-of-truth`.

**Candidate Charter to ratify** (reconstructed — *not* applied anywhere yet):
> Build a reusable **Ladder engine** that compiles and (eventually) **generates**
> quality-gated **learning maps for any goal** — including *describing a codebase as a
> learning map*. The concept graph is the single source of truth; everything derives; a
> gate suite (structural + LLM-judged **correctness** + **craft**) makes generation
> trustworthy. Quality bar = the craft bar (hook → picture/analogy-first → Therefore/But).
> Stance: **learner empowerment + neutrality** over the author's preferred answer. First
> curriculum = the (rescoped) second brain; first dogfood of "describe-a-system" = the
> learning-map-of-the-engine.

If you ratify (or amend) this, it becomes the fixed reference every doc is checked against.
